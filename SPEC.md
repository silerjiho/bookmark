# 포켓몬 박스 — 게임 기획서

이 문서는 포켓몬 박스 앱의 게임 메커니즘 기획을 정리한 것이다.
변경 시 본 문서를 먼저 갱신하고, 코드는 본 문서를 따라간다.

---

## 1. 포획 (Catch)

- **포획 대상**: 전체 포켓몬 종(species) 중 **가장 낮은 진화체** (= base form)만 등장한다.
  - 판정 기준: PokeAPI `pokemon-species` 엔드포인트의 `evolves_from_species === null`.
  - 예) 이상해씨 ✓ / 이상해풀 ✗ / 피츄 ✓ / 피카츄 ✗
  - 더 이상 진화하지 않는 단일 종(예: 켄타로스)도 base form으로 간주하여 등장.
- **선택 방식**: 1 ~ `MAX_SPECIES_ID`(현재 1025) 범위에서 랜덤 ID를 뽑은 뒤 base form이 아니면 재시도(최대 10회).
- **저장**: 포획 결과는 GitHub Issue 1건으로 저장된다 (기존 구조 유지).

## 2. 포켓몬 상호작용

각 포켓몬은 다음 4가지 상호작용을 지원한다.

### 2-a. 나무열매 먹이기 (레벨업)

- 사용 가능한 나무열매는 **고정 3종**:
  - 오랭열매
  - 자뭉열매
  - 유류열매
- 한 번 먹일 때마다 레벨이 **+3 / +5 / +10** 중 하나만큼 오른다.
- 각 포켓몬마다 **세 열매에 대한 선호도(레벨 상승치 매핑)가 다르다**.
  - 즉, 어떤 포켓몬은 오랭=10, 자뭉=5, 유류=3 / 다른 포켓몬은 오랭=3, 자뭉=10, 유류=5 식으로 분배된다.
  - 매핑은 `pokemonId` 기반으로 결정적으로 산출 → 같은 종은 항상 같은 선호도를 갖는다.
  - 가능한 순열은 [3,5,10]의 6가지 중 하나.
- **레벨 상한**: 100. 상한 초과분은 잘려 100으로 고정된다.
- 레벨이 1 이상 오를 때마다 **포인트 +50/Lv** 보상 (아래 2-d 참고).

### 2-b. 놀아주기 (친밀도)

- "놀아주기" 버튼으로 친밀도 상승.
- 친밀도 범위: **0 ~ 5** (`Lv.5`가 max).
- 클릭 1회당 +1, max 도달 시 버튼 비활성화.
- 친밀도는 일부 포켓몬의 진화 트리거로 사용된다(아래 3 참고).

### 2-c. 도구 (Held Item)

- 한 포켓몬은 동시에 **단 1개**의 도구를 장착할 수 있다.
- 도구는 **상점**에서 포인트로 구매한다.
- 이미 도구를 장착 중인 상태에서 신규 구매·장착 시 기존 도구는 **사라진다**(환불 없음).
- 도구는 진화 트리거로 사용되거나, 단순 수집·장식 용도이다.
- 도구 카탈로그는 `src/api/items.ts`에서 관리한다 (모두 진화의 돌 위주).

### 2-d. 포인트 (재화)

- 포인트는 두 가지 경로로 획득된다.
  1. **시간 경과**: 포켓몬을 잡거나 마지막 정산 이후 흐른 시간에 비례 (`POINTS_PER_HOUR = 10`).
     - 박스를 다시 열거나 액션을 수행할 때 `tick`되어 누적된다.
  2. **레벨업 보상**: 레벨이 N 만큼 올랐다면 `+50 × N` 포인트.
- 포인트는 상점에서 도구 구매에 사용한다.
- 모든 도구의 가격은 **100 포인트로 동일**하다.

---

## 3. 진화 (Evolution)

- 진화 정보는 **PokeAPI `evolution-chain`을 그대로 사용**해 트리를 가져온다.
- 한 포켓몬은 0개 이상의 다음 진화 후보를 가질 수 있다 (예: 이브이는 다중 분기).
- 각 진화 후보는 다음 중 하나의 트리거를 가진다:
  - **레벨 진화** (`level-up` + `min_level`)
    - 현재 레벨 ≥ `min_level`이면 진화 가능.
  - **친밀도 진화** (`level-up` + `min_happiness`)
    - 친밀도가 max(=5)이면 진화 가능.
    - PokeAPI의 `min_happiness`(보통 220) 임계값은 본 게임 친밀도 5단계로 단순화한다.
  - **도구 진화** (`use-item` + `item.name`)
    - 해당 도구를 장착(held)한 상태에서 진화 가능.
    - 진화 시 도구는 소모된다(`heldItem` → null).
- 그 외 트리거(거래, 위치, 시간대 등)는 본 게임에서 **지원하지 않으며**, UI에 "지원되지 않는 진화 조건"으로 표시한다.

## 4. 데이터 모델

### `MyPokemon` (런타임 / GitHub Issue 저장 대상)

| 필드                                  | 타입           | 비고                                        |
| ------------------------------------- | -------------- | ------------------------------------------- |
| `issueNumber`                         | number         | GitHub Issue 번호                           |
| `uniqueId`                            | string         | 포획 시점 기반 고유 ID                      |
| `pokemonId`                           | number         | 포켓몬 도감 번호 (진화 시 변경됨)           |
| `name`                                | string         | 한글 종 이름 (계산값)                       |
| `nickname`                            | string         | 별명                                        |
| `level`                               | number         | 1 ~ 100                                     |
| `friendship`                          | number         | 0 ~ 5                                       |
| `points`                              | number         | 보유 포인트                                 |
| `heldItem`                            | string \| null | 도구의 PokeAPI 영문 키 (예: `"fire-stone"`) |
| `image`, `types`, `evolutionChainUrl` | —              | 계산값                                      |
| `caughtAt`                            | ISO string     | 포획 시각                                   |
| `lastTickAt`                          | ISO string     | 시간 경과 포인트 마지막 정산 시각           |

GitHub Issue body 의 JSON 블록에는 위 중 **저장이 필요한 값**만 기록한다:
`{ uniqueId, pokemonId, nickname, level, friendship, points, heldItem, caughtAt, lastTickAt }`

### 후방 호환

- 구버전 Issue (`exp`만 있고 friendship/points/heldItem 없음)는 다음 기본값으로 마이그레이션:
  - `friendship: 0`, `points: 0`, `heldItem: null`
  - `lastTickAt: caughtAt`
  - 기존 `exp`는 무시 (레벨은 그대로 유지).

## 5. UI 변경 사항

- **PokemonCard**: 레벨 + 친밀도 하트 5개 + 보유 도구 아이콘 노출.
- **PokemonDetail**:
  - 레벨 표시 (Lv.N / 100)
  - 친밀도 (하트 5개)
  - 포인트
  - 보유 도구 (있으면 표시 + "해제" 버튼)
  - 나무열매 3종 (각 +레벨치 표기)
  - "놀아주기" 버튼 (max 시 비활성화)
  - "상점" 버튼 → ShopModal
  - 진화 후보 카드들 (각 후보의 조건/충족 여부 표시)
  - "놓아주기" 버튼
- **ShopModal** (신규): 도구 그리드, 가격 100원 표시, 보유 포인트 부족 시 비활성화.

## 6. GitHub Issue 메타데이터 활용

GitHub Issue는 단순 저장소가 아니라 게임 메타데이터 채널로도 사용한다.

### 6-a. Label — 세대(Generation)

- 각 포켓몬 Issue에 **소속 세대 1개**를 라벨로 부착한다.
- 라벨 이름: `1세대`, `2세대`, … `9세대`.
- 세대는 PokeAPI `pokemon-species.generation.name` (`generation-i` ~ `generation-ix`)에서 추출.
- 진화해도 세대는 변하지 않으므로 포획 시점에만 부착하면 된다.
- 라벨이 저장소에 없으면 자동으로 생성한다 (color: `ededed`).

### 6-b. Milestone — 주 타입(Primary Type)

- 각 포켓몬 Issue에 **주 타입 1개**를 마일스톤으로 부착한다.
- 마일스톤 제목: `노말`, `불꽃`, `물`, … (한글 타입명, `TYPE_KO` 사용).
- 2 타입을 가진 포켓몬은 `types[0]` (PokeAPI slot 1)을 사용.
- 진화 후 주 타입이 바뀌면 마일스톤도 새로 부착한다.
- 마일스톤이 저장소에 없으면 자동으로 생성한다.

### 6-c. Comment — 이벤트 로그

다음 두 이벤트 발생 시 Issue에 코멘트를 자동으로 남긴다.

| 이벤트                               | 코멘트 본문                          |
| ------------------------------------ | ------------------------------------ |
| 진화                                 | `{별명}는 {새이름}로 진화했다!`      |
| 친밀도 max 도달 (5단계 첫 진입 순간) | `{별명}는 트레이너와 아주 친해졌다!` |

- 친밀도 코멘트는 `0~4 → 5` 전이 1회만 발생하며, 이미 5인 상태에서 추가 발생하지 않는다.
- 진화 코멘트는 매 진화마다 발생.

## 7. 비동기 UX — "포켓몬을 전송중" 오버레이

GitHub Issues API CRUD는 체감상 느리다 (라벨/마일스톤 사전 보장 + 코멘트 동시 작성 시 더욱).
화면에 반영된 데이터가 보이기 전까지는 사용자가 다른 동작을 시도하지 못하도록,
**전역 전송중 오버레이**를 띄운다.

- 트리거 대상 (모든 GitHub 쓰기/생성 동작):
  - 포획, 나무열매 먹이기, 놀아주기, 도구 구매, 도구 해제, 진화, 놓아주기.
- UX 사양:
  - 전체 화면을 가리는 반투명 오버레이.
  - 중앙에 메시지 "포켓몬을 전송중...".
  - 모든 클릭 차단 (포인터 이벤트 캡처).
- 오버레이가 표시되는 동안 모든 액션 버튼은 disabled 상태와 동일하게 동작한다.
- 상태는 `App.tsx`에서 단일 `transferring` 변수로 관리하고, 각 핸들러는 시작 시 true / 종료 시 false 로 토글한다.

---

## 8. 비기능 요구사항

- 외부 호출은 PokeAPI (`pokemon`, `pokemon-species`, `evolution-chain`)와 GitHub Issues API 두 가지뿐이다.
- 나무열매와 도구는 **정적 데이터**(코드 내 상수)로 보유한다.
- 레벨·친밀도·포인트 계산은 모두 클라이언트 단에서 결정적으로 수행한다.
- 라벨/마일스톤은 모듈 레벨 캐시로 중복 호출을 줄인다.
